# QareebShare Android — قريب شير

تطبيق Android أصلي وخفيف لنقل الملفات والتطبيقات بين هاتفين قريبين دون إنترنت أو راوتر.

## الحالة الحالية

اكتملت الدفعات البرمجية الأساسية حتى الإصدار `0.4.0`:

- Java وXML أصليان.
- دون AndroidX أو Compose أو Kotlin أو أي مكتبة تشغيل خارجية.
- Android 6 (API 23) حتى Android 17 (API 37).
- Wi-Fi Direct لإنشاء الرابط بين الهاتفين.
- TCP Socket محلي لنقل البيانات.
- اختيار ملفات متعدد ومشاركة مباشرة من تطبيقات أندرويد الأخرى.
- عرض التطبيقات المثبتة غير التابعة للنظام.
- دعم Base APK وSplit APKs.
- رمز مطابقة يظهر على الهاتفين.
- ECDH P-256 ومفاتيح جلسة مؤقتة.
- تشفير AES-GCM للبيانات والملفات.
- نقل Streaming بذاكرة ثابتة.
- استكمال النقل بعد الانقطاع.
- SHA-256 للتحقق من سلامة الملف الكامل.
- حفظ النتائج في `Downloads/QareebShare`.
- تثبيت التطبيقات المستلمة عبر PackageInstaller بعد موافقة المستخدم.
- سجل SQLite محلي خفيف.
- إشعار تقدم وخيار إلغاء النقل.
- بناء GitHub Actions يدوي فقط.

## بيئة البناء

- Android Gradle Plugin: 9.2.0
- Gradle: 9.4.1
- JDK: 17
- SDK Build Tools: 36.0.0
- compileSdk / targetSdk: 37
- minSdk: 23

## الفحص دون بناء

```bash
./scripts/check-final.sh
```

## البناء من الهاتف

بعد اكتمال الدفع إلى GitHub:

1. افتح المستودع من المتصفح أو تطبيق GitHub.
2. افتح `Actions`.
3. اختر `Android Build`.
4. اضغط `Run workflow`.
5. بعد النجاح نزّل `qareeb-share-apks` من Artifacts.

البناء لا يعمل تلقائياً عند `git push`.

## الخصوصية

لا تُرفع الملفات إلى أي خادم، ولا يتطلب التطبيق حساباً، ولا توجد تحليلات أو إعلانات أو خدمات سحابية.
