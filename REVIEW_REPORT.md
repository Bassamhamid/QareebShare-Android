# تقرير مراجعة مشروع قريب شير

## عيوب تم تصحيحها

1. فحص الإصدار كان مربوطاً حرفياً بالإصدار 0.4.0، لذلك فشل بعد رفع المشروع إلى 0.5.0.
2. أوامر التصحيح السابقة أدخلت مسافات زائدة في نهاية سطرين، فأوقف `git diff --check` العملية.
3. Workflow ظل يستخدم فحصاً قديماً لا يعرف إعادة البناء.
4. رفع APK كان مضبوطاً على `if: always()`، لذلك حاول رفع ملفات غير موجودة بعد فشل مبكر.
5. إصدارات GitHub Actions القديمة كانت سبب تحذير Node 20.
6. منطق الاتصال كان يغيّر الحالة إلى CONNECTING ثم يزيل المجموعة القديمة. إشعار الانقطاع الناتج عن الإزالة كان يعيد الحالة إلى IDLE، فتتوقف النقرة بلا اتصال.
7. الاتصال اعتمد على Broadcast واحد فقط ولم يستطلع `requestConnectionInfo` بعد نجاح `connect()`.
8. تنظيف Wi‑Fi Direct اعتمد على تأخيرات ثابتة بدلاً من انتظار callbacks الفعلية.
9. اختيار «حسب النظام» في الإصدارات القديمة لم يكن يعيد المظهر دائماً إلى حالة النظام بعد اختيار فاتح أو داكن.
10. صف الجهاز غير المتاح لم يكن يمنع ListView من استدعاء النقر بشكل موثوق.

## فحوص نُفذت

- تحليل جميع ملفات XML.
- مطابقة مراجع النصوص والأنماط والألوان.
- فحص Bash لكل السكربتات.
- فحص المسافات الزائدة في كل ملفات المشروع النصية.
- ترجمة ملفات الاتصال المعدلة بواسطة `javac` مع واجهات Android وهمية للتحقق من ترابط Java.
- فحص عدم وجود مكتبات تشغيل خارجية.

لم يُبنَ APK داخل بيئة المراجعة؛ البناء الفعلي يجب أن يتم في GitHub Actions، ثم اختبار الاتصال على هاتفين.
